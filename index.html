一、项目背景

雀巢希望借助小河马线上商城，由母婴门店帮助代销"佳膳优选"直供商品。消费者在小河马商城线上下单，雀巢大仓直发快递给消费者，门店获得 20% 佣金。

佣金直接打给门店个人不合规，需借助中台 → 达生的任务办单链路，以积点形式发放佣金，门店通过达生提现。

本次仅涉及小河马商城订单的正向积点结算和逆向退货扣减，不涉及资金预充值。

二、业务流程概述

本系统涉及四条核心链路，按执行先后顺序排列：

序号链路触发方说明
① 任务初始化 达生 达生创建长期任务（至年底）→ 通知中台 → 中台维护 SKU 与门店 → 推送小河马 → 账号同步 → 自动领取
② 正向下单 → 积点结算 消费者 小河马商城下单支付 → A10 推送中台 → 渠道分流（跳过原中台任务校验）→ 推送达生 → 按"任务+产品"基点发积点
③ 逆向退货 → 积点扣减 消费者 小河马商城退货 → A10 推送退货单 → 中台记录 NHS → 同步达生 → 达生扣减待发放积点
④ 大仓发货（旁路） 雀巢大仓 大仓根据订单直发快递给消费者，与积点链路解耦

六、中台调整范围

1️⃣ 正向链路

1. 配额相关

⚠️ 待定 配额方案待讨论确认。如需限制大区配额和门店配额，需在现有配额配置页面增加任务编号关联，按任务维度配置配额。

2. A10 门店任务售出办单数据同步接口 — 字段变更

字段原状态新状态说明
channelType（业务类型） — 🆕 新增 枚举：XHM_MALL（小河马商城）/ TASK_REDPACKET（任务红包）
transactionType（交易类型/订单类型） — 🆕 新增 枚举：SALE（销售单）/ RETURN（退货单）
source_code（溯源码） 必填 🆕 改为非必填 小河马商城订单无溯源码，该字段改为非必填

3. 小河马商城渠道处理逻辑

当 channelType = XHM_MALL 时：

  • 中台不进行原有的任务逻辑校验
  • 直接以有效状态通过 A12-中台推送消费者开罐扫码数据接口推送达生
  • 内码的扫码数据不传
  • source_code 需生成一个特殊标识开头的随机字符串，标记为虚拟生成的溯源码（建议由小河马侧生成提供）

2️⃣ 逆向链路

1. 逆向退货流程

  • 逆向链路为新增流程，仍复用 A10-门店任务售出办单数据同步接口
  • 当 transactionType = RETURN 时，中台接收小河马的退货订单数量
  • 中台进行基础校验：
    • 校验销售单号在中台是否存在
    • 校验退货SKU是否存在于原销售单中
    • 退货数量不能大于该笔订单的销售数量
    • 校验该笔订单是否已全额退货（防止重复退）
  • 中台接收到退货订单后，存入单独的退货记录表，并与售出单进行关联

2. 通知达生退货处理

中台通过 A12-中台推送消费者开罐扫码数据接口通知达生进行退货单处理，其中产品售出数量为负数。

七、识别的流程风险

风险点所在图说明建议
R1：达生仅以外码判定积点的兼容性 图4 中台对 XHM_MALL 订单：外码附随机值+标记有效，内码不处理不传达生。达生仅以外码状态判定积点发放是否可行？ 需与达生确认：仅接收外码随机值+有效状态，不接收内码，达生校验是否通过
