一、项目背景

雀巢希望借助小河马线上商城，由母婴门店帮助代销"佳膳优选"直供商品。消费者在小河马商城线上下单，雀巢大仓直发快递给消费者，门店获得 20% 佣金。

佣金直接打给门店个人不合规，需借助中台 → 达生的任务办单链路，以积点形式发放佣金，门店通过达生提现。

本次仅涉及小河马商城订单的正向积点结算，不涉及资金预充值。

二、业务流程概述

本系统涉及两条核心链路，按执行先后顺序排列：

序号链路触发方说明
① 任务初始化 达生 达生创建按月任务 → A1 推送 🆕 任务类型 到中台 → 中台维护 SKU 与门店（非SFE归虚拟大区）→ 配额增加 🆕 任务类型维度 → B3 推送小河马（含 🆕 任务类型）→ 账号同步 → 首次手动领取
② 正向下单 → 积点结算 消费者 小河马商城下单支付 → 确认收货生成虚拟罐底码 → A10 推送中台 → 跳过校验 → A12 推送达生 → 按"任务+产品"基点发积点

五、中台调整范围

中台改造内容

1. 新增「任务类型」字段

1.1 方案概述

采用任务类型（taskType）字段，在现有任务-配额链路中增加区分维度。中台数据库已有该字段，无需新增数据库字段。

📌 核心变化 达生创建任务时通过 A1 接口推送任务类型，中台接收并存储，不再由中台侧手动维护。

1.2 新增任务类型的接口清单（4个）

接口编号接口名称方向改动内容影响方
A1 任务信息同步 达生 → 中台 达生创建任务后将任务主数据同步给中台，新增 任务类型 字段 达生侧配合改造
B2 大区配额同步 中台 → 达生 配额推送时新增 任务类型 字段，达生据此区分不同任务的配额扣减 达生侧配合改造
B1 门店配额同步 中台 → 达生 配额推送时新增 任务类型 字段，达生据此区分不同任务的配额扣减 达生侧配合改造
B3 任务信息推送 中台 → 小河马 中台将任务数据推送给小河马，新增 任务类型 字段，小河马据此区分不同任务并关联正确任务编号 小河马需区分不同任务类型
📌 不新增任务类型、但逻辑受影响的接口
接口编号接口名称方向说明
A2任务门店范围推送中台 → 小河马门店范围推送时需包含虚拟大区门店
A3门店信息同步小河马 → 中台小河马根据推送的门店编码回传虚拟大区门店数据
A10门店任务售出办单数据同步小河马 → 中台办单推送不改动；中台推送达生时不传任务类型，达生通过任务编号自行查询

2. 配额拆分

2.1 配额维度改造

大区配额和门店配额均新增🆕任务类型维度，同一大区/门店可按任务类型拆成多条配额记录，互不挤占。现有配额配置页面调整，支持按 🆕 任务类型 维护配额。

配额类型是否必填说明
外码配额 ✅ 必填 门店办单出库扫描罐底码消耗，本次小河马佳膳优选活动仅需配置外码配额
内码配额 ⚠️ 非必填 消费者扫码参与营销活动消耗，本次小河马佳膳优选活动不需要配置内码配额

示例：某门店同时参与两类任务

门店任务类型外码配额内码配额
门店 A扫码红包8080
门店 A小河马佳膳优选50—（不填）

3. 非 SFE 架构门店处理

3.1 门店归类方案

场景处理方式
门店在 SFE 架构内 按现有逻辑，保留原有大区信息（东/西/南/北/中）
门店不在 SFE 架构内 需先在后台维护该门店信息，并将大区赋值为虚拟大区编码（名称待定），纳入配额体系

涉及接口：A2（中台→渠道，门店范围推送含虚拟大区门店）、A3（渠道→中台，小河马回传虚拟大区门店数据）。虚拟大区仅作为兜底归类，不与真实 SFE 架构冲突。

4. A10 接口字段变更

字段原状态新状态说明
source_code（溯源码/罐底码） 必填 必填（不变） 保持必填；小河马自行生成虚拟罐底码传入

5. 任务有效期

新任务沿用现有逻辑：按月推送，首次需手动领取，后续自动领取。配额按月维护，与现有扫码红包体系一致。

6. 小河马侧配合内容

  • 交易成功后才推送办单数据到中台 A10 接口
  • 自行生成虚拟罐底码，编码规则：HM + 年 + 点 + 8位随机数（示例：HM26.06X17OO2），不与雀巢现有罐底码规则冲突
  • A10 推送时 source_code 字段使用该虚拟罐底码
  • 接收 B3 推送的 🆕 任务类型 字段，用于区分不同任务并关联正确任务编号
📌 完整办单数据流 达生按月创建任务(A1+🆕任务类型) → 中台维护SKU/门店/配额(🆕任务类型维度) → B3推送小河马(含🆕任务类型) → 小河马下单 → 确认收货生成虚拟罐底码 → A10推送中台 → 跳过校验 → A12推送达生 → 达生按任务编号查任务类型 → 计算基点 → 门店佣金结算。

逆向流：暂时不考虑。如有线下退货，由门店自行处理，不进入中台-达生链路。
